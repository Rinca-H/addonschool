minecraft:frictionDec0.0
このブロックにかかる摩擦の量・方式。
数値の範囲効果
0.1 ～ 0.5数値が小さいほどスピードポーションのように加速するようになり、大きければソウルサンドのように減速する(0.1未満は使用不可)
0.6通常の摩擦。この値を指定するぐらいならこのコンポーネントは必要ない。
0.7 ～ 1.0数値が大きいほど氷の上のように長く滑る。
1.1移動速度が減衰せず、歩けば歩くほど加速する。
1.2 ～歩かなくとも加速する、等加速度運動。

render_methodStr"opaque"このブロックのレンダー方式。"opaque"、"alpha_test"、"blend"、"double_sided"から選べる(*1)。

名前備考
opaqueテクスチャの透明度を無視してすべて不透過にする。完全に透明な部分(=色がない部分)は黒になる。
alpha_testテクスチャの完全に透明な部分のみを透明にする。
blendテクスチャの透明度に応じて透明にしたり半透明にしたり不透明にしたりする。
double_sided基本はopaqueと同じ。ただしブロックの内側にも描画する(opaqueの場合、ブロックの一面を透明にして不透明な面の内側を見ると透けてしまう)。

minecraft:placement_filterObj　
名前型備考
conditionsArray(Obj)
名前型備考
allowed_facesArray(Str)どの面に接するように置けるか。どの面に接していれば残っていられるか。"up"(上面)、"down"(下面)、"north"(北面)、"south"(南面)、"east"(東面)、"west"(西面)、"side"(横面)、"all"(全面)のうちから一つ以上選択。面を指定しない場合はこのプロパティ１行分必要ない。upのみを指定すれば他のブロックの上面にしかこのブロックを置けなくなる。
block_filterArray(Str)どのブロックに接して置けるか。どのブロックに接していれば残っていられるか。idでいくつか指定。ブロックを指定しない場合はこのプロパティ１行分必要ない。

たとえば、草ブロックと土の上面と石の北面と南面にのみ置けるようにするならコンポーネント全体はこのようになる。 
"minecraft:placement_filter": {
  "conditions": [
    {
      "allowed_faces": [ "up" ],
      "block_filter": [ "minecraft:dirt", "minecraft:grass" ]
    },
    {
      "allowed_faces": [ "north", "south" ],
      "block_filter": [ "minecraft:stone" ]
    }
  ]
}

minecraft:unit_cubeObj　テッセレーションを行うユニットのキューブを指定します。本来テッセレーションとは、3Dゲームにおいて球などの曲面に対して、そのオブジェクトに近づいたときはより滑らかに、離れたときは処理を軽減するために大まかにレンダリングする(ために使う方法の)ことですが、あってもなくても分かりやすい違いはないので基本はなくていいと思います。
