　パックが作れないとリソースパック、またはビヘイビアパックとしてそもそも導入できないので必ず知ってください。

基礎行程１

　まずは空(から)のパックを作ってみましょう。

①入れ物を用意する

　まずはアドオン専用の新規フォルダを作りましょう。フォルダの名前は自由です。今回は最低限必要なもの以外何も入れませんが、アドオン作りは基本そのフォルダ内で行われることになります。たくさん作るときはビヘイビアパックやリソースパックなどのパック一つごとに一つフォルダを増やす感じになります。

②パックのアイコンを用意する

　パックのアイコンとは、ゲーム内の設定でアドオン選択画面に表示されるアイコンです。サイズの指定は特にないと思います。アイコンの画像を用意したら、ファイルの名前は必ず「pack_icon」にしてください。拡張子は「.png」で統一しましょう。もし用意した画像の拡張子が「.jpg」や「.jpeg」で「.png」に変えようとしても勝手に書き換えてはいけません。変換するか、PNG形式で保存し直してください。変換する場合は「jpg png 変換」などと検索すればオンラインコンバータが簡単に見つかると思います。
　用意したものは、はじめに作ったフォルダに置いてください。
　面倒ならテンプレートに入っている「pack_icon.png」をそのまま使用しても構いません。

③マニフェストを用意する

　マニフェストとは、パックをインポートしたときにこれらのファイル群がアドオンを構成するものだとゲーム本体に認識させるためのファイルです。
　イチから自分で書くのは面倒なので、テンプレートに同封されている「manifest.json」のコピーから必要な部分だけ書き換えていきます。これをコピーして、はじめに作ったフォルダに貼り付けます。今回は中身がないのでビヘイビアのほうからコピーしてもリソースのほうからコピーしても構いません。大差ないですができるだけ作りたいパックのほうからコピーするといいでしょう。

④マニフェストを編集する

　マニフェストをテキストエディタで開きましょう。「.json」が名前にある「JSONファイル」は既定で(常時)そのテキストファイルで開くように設定するといいでしょう。 

{
    "format_version": 2,
    "header": {
        "description": "Example vanilla behavior pack",
        "name": "Vanilla Behavior Pack",
        "uuid": "ee649bcf-256c-4013-9068-6a802b89d756",
        "version": [ 0, 0, 1 ],
        "min_engine_version": [ 1, 16, 210 ]
    },
    "modules": [
        {
            "description": "Example vanilla behavior pack",
            "type": "data",
            "uuid": "fa6e90c8-c925-460f-8155-c8a60b753caa",
            "version": [0, 0, 1]
        }
    ],
    "dependencies": [
        {
            "uuid": "66c6e9a8-3093-462a-9c36-dbb052165822",
            "version": [0, 0, 1]
        }
    ]
}

　これはビヘイビアのほうから持ってきました。リソースの場合、少し短いと思います。上から順に説明していきます。気にしなくていいものは飛ばしますが、何かで意味を知らない限りは変更しないでください。

○description

　パックの説明文。

○name

　パックの名前。

○uuid

　パックのid。これはパックによって変える必要があり、すでに持っているパックとidがカブっているものをインポートしようとするとエラーになってしまいます。
　そこで、そのidは「Online UUID Generator」で発行してもらいます。発行してもらうには、このサイトに飛んですぐに表示されているUUIDの右側にあるCopyボタンを押せばクリップボードにコピーされるので、あとはもとあったUUIDと入れ換えるだけです。あと、ページを更新すれば新しいUUIDが発行されます。
　まれに、自分がすでに持っているパックとたまたまUUIDがカブってエラーになることがあるので、導入して一度「重複したパックを検出」と出たらUUIDを変更してみてください。(その際パックが上書きされたり消えたりすることはないです)

○version

　パックのバージョン。自分で決められます。

○min_engine_version

　このパックが使えるマインクラフト本体の最低バージョン。いまこのパックを作っているときのバージョン、つまり自分がこのパックの動作確認した最低バージョンでいいと思います。

○type

　パックのタイプ。ビヘイビアなら「"data"」、リソースなら「"resources"」です。先の分野の話になるので聞き流してもいいですが、ScriptingAPIを使用する際のビヘイビアは「"client_data"」となります。テンプレートからコピーしてくる段階でビヘイビアを作る場合はビヘイビアから、リソースを作る場合はリソースからコピーしてくることによってここを書き換える必要がなくなります。

　二度目のdescriptionとversionも同じです。ただし二度目のuuidはまた新しいUUIDに書き換えてください。よって一つのマニフェストを作るのに新しいUUIDを最低で(・・・)二回発行する必要があります。

　最後に少し難しいことをします。下のほうにある「dependancies」のカタマリはややこしいので消します(リソースのほうにはもともとありません)。これは任意であってもなくてもいいので詳しくは「依存関係」の章で説明するのでそちらを読んでください。

○マニフェストは以上

　編集例をあげます。UUID、パックの名前・説明は各自で異なるはずです。
・UUIDは二つ(・・)別のものに書き換えましたか？
・「"」を消したりしていませんか？
・「,」を余分に消していませんか？
・カッコの組は正しい組ですか？

{
    "format_version": 2,
    "header": {
        "description": "これは空(から)のパックです",
        "name": "マイテストパック",
        "uuid": "ef805b52-e547-466f-a2b8-17d045d4726f",
        "version": [ 0, 0, 1 ],
        "min_engine_version": [ 1, 16, 210 ]
    },
    "modules": [
        {
            "description": "これは空(から)のパックです",
            "type": "data",
            "uuid": "f3d885d5-501f-4ba6-848a-0fbe62f134b5",
            "version": [0, 0, 1]
        }
    ]
}

⑤圧縮する

　現在のファイル構成は以下のようになっているでしょうか？

(はじめのフォルダ)┳pack_icon.png
 　　　　　　　　  ┗manifest.json

　右の二つのファイルを複数選択ですべて選択し、.zipファイルに圧縮します。圧縮してできたファイルの拡張子を「.zip」から「.mcpack」に書き換えます。これは特に変換などをせずに書き換え、「ファイルが使えなくなる可能性があります」と言われても気にする必要はありません。
　あと、圧縮した際に、圧縮する前のファイルは残ると思うんですが、近いうちにこのパックに変更を加える場合それらを消す必要はありません。今後パックの中身を更新するときに圧縮したものをわざわざ展開せずに作業が始められます。

　これで中身がない「ビヘイビアパック」または「リソースパック」が完成しました！

　あとは完成したファイルをダブルクリックやタップで開きます。その際、開くプログラムを選ぶことになると思うので、マインクラフトで常時開くように選択します。
　正しくインポートできたら完了です！

　
基礎行程２

　今度は中身を入れることを想定します。「基礎行程１」で作ったものから変更を加えていきます。言い換えると、パックのアップデートのやり方ですね。

①古いパックを消す

　一度ゲーム本体にインポートしたパックは消しましょう(ゲーム内にあるもののみですよ！　元のファイルを消せばもちろんただの作り直しです！)。たとえパックのアップデートであろうとUUIDが同じなのでインポートしてくれません。かといってアップデートのたびに新しいUUIDを発行してインポートすると当然別のパックだと認識されるのでどんどんたまっていきます。


「(タイトル画面)→設定→ストレージ」から削除します。
消すのは.mcpackなどの「本体」ではありませんよ？

②中身に変更を加える

　どのフォルダにどのファイルを入れるかはそれぞれの分野で説明します。ちなみにすべての分野で追加や変更をする場合、このようなフォルダ構成になります。

③圧縮する

　あとは「基礎行程１」と同様です。

　ここまでが基本です。次の章からは必須ではないので任意で読んでみてください。

アドオンパックの作り方

　ビヘイビアパックでオリジナルのコンテンツを追加する際に、リソースパックも同時に必要になってくることがほとんどです。ビヘイビアパックとリソースパックをそれぞれ作って二つのパックをインポートする必要がありますが、二つのパックをインポートするためには、一つのパックをインポートしてからもう一つのパックをインポートしなくてはならないので面倒です。そこで、二つ以上のパックを同時にインポートする方法があります。
　同時にインポートしたい.mcpackファイルを複数選択ですべて選択し、さらに圧縮します。できた.zipファイルの「.zip」を「.mcaddon」に書き換えます。このファイルも.mcpackファイルと同様に開くとマインクラフトが起動してインポートが始まります。インポートは一つずつ行われ、一つのインポートが終わると自動で次のパックのインポートが始まります。

依存関係

　「依存関係」を利用して、ゲーム内のアドオン設定画面でビヘイビアパックを適用したときに自動でセットのリソースパックを適用してくれるようにすることができます。
